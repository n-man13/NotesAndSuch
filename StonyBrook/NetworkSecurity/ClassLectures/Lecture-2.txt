Need for Layered Architecture•Brought about because of a need for standards•Layers:•Take information from above (layer N-1)•and pass information below (layer N+1)•The services are provided through the service access points (SAPs)•Layer functionality is implemented through an entity•Each layer contains one or more entities that are responsible for providing services to the N+1 layer

Layered Architecture (logical & physical channels)•In order for layers to carry out functions, they need to communicate•A layer N entity may need to communicate with another layer N entity, which does not reside on the same system, to provide the service.•The layer N entity uses the layer N-1 services to communicate with the remote layer N entity.

What are Protocols?•PROTOCOLS are the rules/discipline that have been defined for the layer N to layer N communication.•They represent extra information•Example: saying “hello” on the telephone is a protocol•Protocols indicate when to send data, what language to use, etc.•A layer specification defines•What protocol does it use?•What does it expect as input (SAPs)?•What functions does it provide?•Layer specifications allow multiple vendors to have the same functionality.•Example: Different ethernet card brands

Basic Functions of a Protocol in Networking1.Segmentation and reassembly 2.Encapsulation3.Connection control4.Ordered delivery5.Flow control 6.Error control7.Multiplexing

Segmentation and reassembly•Often, physical media or error control issues dictate a maximum data size.•Generically, this size is called Maximum Transmission Unit (MTU)•Therefore, the data must be divided into smaller packets (Segmentation)•And eventually put back together (Reassembly)•Reassembly instructions are included in the header

Encapsulation•Encapsulation is the addition of control information to the data element in the form of a header.•Address: The address of the sender and/or receiver.•Error Detection Code: Some sort of code is often included for error detection.•Protocol Control: Additional information that is needed to implement the protocol.

Connection Control•Connectionless Data Transfer•Data is transferred without prior coordination•No set path•Connection-oriented Data Transfer•A logical association, or Connection, is established between entities before any data is transferred•Example: telephone•Three phases of Connection Control•Request/connect phase •Data transfer phase •Termination phase

Connection Control•Connectionless Data Transfer•Data is transferred without prior coordination•No set path•Connection-oriented Data Transfer•A logical association, or Connection, is established between entities before any data is transferred•Example: telephone•Three phases of Connection Control•Request/connect phase •Data transfer phase •Termination phaseConnection can be viewed as a communication state

Ordered Delivery• Pieces arrive in the same order as sent• Not provided by connectionless protocols• Not required to be provided by Connection-oriented protocols, but it is common for most. (needed for file transfer)

Flow Control•Technique for assuring that the transmitting entity does not overwhelm a receiving entity.•Flow Control is typically implemented in several layers.• Sometimes, the goal of flow control is to ensure fairness.•Flow control is found in most connection-oriented protocols.

Error Control•A technique that allows a protocol to recover from lost or damaged PDUs.•Three mechanisms:•Positive acknowledgment•Retransmit after timeout•Error detection

Multiplexing•Upward Multiplexing occurs when multiple higher-level connections are multiplexed on a single lower-level connection. •Example: Many applications (telnet, FTP , email) utilize TCP •Downward Multiplexing occurs when a single higher-level connection is multiplexed on multiple lower-level connections. (not as common)•Addressing is needed to support multiplexing.

OSI Model of Network•Application•Presentation•Session•Transport•Network•Data Link•Physical

Physical Layer•Responsible for the transparent transmission of bit streams across the physical interconnection of systems •Two configurations:•Point-to-point•Multipoint•Physical layer must provide the data link entities with a means to identify the end point.•Physical connection can be Full Duplex or Half Duplex•Physical connection can be either bit serial or N bit parallel•Physical layer must deliver the bits in the same order in which they were offered for transmission by the Data Link Layer.

Data Link Layer•The main task is to shield the higher layers from the characteristics of the physical transmission medium. •It should provide the higher layers with a reliable transmission that is basically Error-Free, although errors may occur in the transmission on the physical connection.•Services provided should be independent of the data transmitted.•The data link layer connects two network entities in adjacent systems called Data link connections.

Data Link Layer (contd.)•Each data unit from the network layer is mapped into the data link protocol data unit along with the data link protocol information and is called a Frame.•The data link layer must provide a method of recognizing the start and end of the Frame.•Frames must be presented to the network in the same order they are received.•The data link layer should also implement Flow Control to prevent data overrun.

Network Layer•The primary responsibility of the network layer is to provide the transparent transfer of all data submitted by the transport layer to any transport entity anywhere in the network.•The network layer must handle the routing of data packets.•The network layer can be the highest layer in a device, such as a gateway or router.•IP protocol

Transport Layer•Responsible for a reliable transparent data transfer between two session layer entities.•Transport connection is provided to the session entities independent of their location.•The transport layer must optimize resources while maintaining a guaranteed quality of service.•The session layer requests a level of service, and once the transport connection is provided with a certain quality of service, it must be maintained unless it is notified of the change.•TCP protocol (connection-oriented), UDP (connection-less)

Transport Layer (Contd.)•The transport layer is only concerned with the transfer of data between session layers. It is not aware of the structure of the underlying layers or the topology.•The transport layer uses the network layer to establish a network connection from one transport entity to another.•Depending on the quality of the network, the transport layer may have to perform additional functions to offer the service.•The transport layer provides flow and error control.

Session Layer•The session layer is not concerned with the network.•The session layer’s goal is to coordinate the dialog between presentation layers•The session layer must provide the establishment of a session connection and the management of the dialog on that connection.•Example: An atm maintains a constant connection with a bank (transport service). The session starts when the user logs in.

Presentation Layer•The presentation layer provides the application layer with services related to the presentation of information in a form that is meaningful to the application entities.•The presentation layer provides the mechanism for the application layer to translate its data into a common format that can be translated by the peer application entity.•Marshalling and Unmarshalling•Data format M1 à Network format à Data format M2 

Application Layer•The highest layer provides a means for application processes to access the OSI stack.•Provides both general services and application-specific services.•This is what the user sees and the developers interact with in most cases•Examples: telnet, ftp, web

Network Management, Monitoring, and Security•When the layered architecture was designed, little to no attention was given to network management, monitoring, and security.•These services were not considered critical when networks were small and controlled only by a few organizations. •With the growing size and complexity of networks and the Internet, the need for these services became critical•Looking through the services, it became clear that the services require access to all the layers of the network to make reasonable decisions.•Due to these observations, a new architecture had to be considered where layered and non-layered services are combined.
